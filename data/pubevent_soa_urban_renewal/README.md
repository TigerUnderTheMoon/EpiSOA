# PubEvent-SOA Urban Renewal Dataset

This directory is the working layout for the paper-grade urban-renewal PubEvent-SOA dataset.

The intended scale is 40-60 public urban-renewal events. Build it incrementally: first run the full pipeline on 3-5 events, then expand after validation and annotation agreement are stable.

## Directory Layout

- `raw/raw_posts.jsonl`: imported or lightly crawled public posts/comments/articles.
- `intermediate/candidate_evidence_pairs.jsonl`: annotation-ready emotion-cause evidence-pair candidates.
- `silver/silver_tuples.jsonl`: weak labels generated by rules or LLMs. Silver data is never gold evaluation data.
- `annotations/`: annotation sheets, annotator files, adjudication files.
- `gold/events.jsonl`: finalized event definitions.
- `gold/evidence.jsonl`: human-reviewed evidence pairs used by EpiSOA.
- `gold/gold_tuples.jsonl`: human-verified gold attribution tuples.
- `gold/gold_event_chains.jsonl`: human-verified event-chain labels.
- `reports/`: cleaning, agreement, validation, and dataset statistics reports.

## Recommended Workflow

```powershell
python scripts/import_raw_posts.py --input input.csv --output data/pubevent_soa_urban_renewal/raw/raw_posts.jsonl
python scripts/extract_events.py --raw-posts data/pubevent_soa_urban_renewal/raw/raw_posts.jsonl --output data/pubevent_soa_urban_renewal/gold/events.jsonl
python scripts/generate_silver_tuples.py --raw-posts data/pubevent_soa_urban_renewal/raw/raw_posts.jsonl --events data/pubevent_soa_urban_renewal/gold/events.jsonl --output data/pubevent_soa_urban_renewal/silver/silver_tuples.jsonl
python scripts/build_evidence_pairs.py --raw-posts data/pubevent_soa_urban_renewal/raw/raw_posts.jsonl --silver data/pubevent_soa_urban_renewal/silver/silver_tuples.jsonl --output data/pubevent_soa_urban_renewal/intermediate/candidate_evidence_pairs.jsonl
python scripts/build_annotation_sheet.py --events data/pubevent_soa_urban_renewal/gold/events.jsonl --evidence data/pubevent_soa_urban_renewal/gold/evidence.jsonl --candidate-pairs data/pubevent_soa_urban_renewal/intermediate/candidate_evidence_pairs.jsonl --output data/pubevent_soa_urban_renewal/annotations/annotation_sheet.csv
```

After human annotation and adjudication, convert and validate:

```powershell
python scripts/convert_annotation_csv_to_gold.py --input data/pubevent_soa_urban_renewal/annotations/annotation_sheet_filled.csv --output data/pubevent_soa_urban_renewal/gold/gold_tuples.jsonl --validation-output data/pubevent_soa_urban_renewal/reports/dataset_validation_report.json
python scripts/compute_agreement.py --annotator-a data/pubevent_soa_urban_renewal/annotations/annotator_a.jsonl --annotator-b data/pubevent_soa_urban_renewal/annotations/annotator_b.jsonl --output data/pubevent_soa_urban_renewal/reports/agreement_report.json
```

## Rules

- Do not bypass logins, CAPTCHAs, paywalls, or platform restrictions.
- Do not put direct personal identifiers in `gold/evidence.jsonl`.
- Keep `silver_tuples.jsonl` separate from gold data and mark it with `label_source="llm_silver"`.
- Gold tuples must include `support_label` and must reference existing evidence IDs.
