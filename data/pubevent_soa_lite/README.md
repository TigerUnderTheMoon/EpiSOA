# PubEvent-SOA Lite Data Directory

This directory is an empty event-first data skeleton. Formal data collection starts from accepted concrete public events in `events.jsonl`.

## Files and Directories

```text
events.jsonl
raw/
interim/
annotation/
evidence.jsonl
gold_tuples.jsonl
gold_event_chains.jsonl
```

## events.jsonl

`events.jsonl` is the formal event registry. It must contain accepted concrete public events only.

Each event should include:

- `event_id`
- `domain`
- `event_type`
- `event_name`
- `event_description`
- `location`
- `time_window`
- `trigger`
- `anchor_entities`
- `anchor_urls`
- `source_scope`
- `query_seeds`
- `stakeholder_hints`
- `stance_hints`
- `temporal_stages`

Validate it with:

```bash
python scripts/validate_events.py
```

## raw/

`raw/` stores generated raw posts from evidence collection. It is ignored by git except for `.gitkeep`.

Generate it with:

```bash
python scripts/collect_evidence.py
```

## interim/

`interim/` stores generated query plans, coverage reports, deduplication reports, and other temporary data products. It is ignored by git except for `.gitkeep`.

## annotation/

`annotation/` stores generated evidence annotation sheets, LLM preannotation outputs, review sheets, and inspection samples. It is ignored by git except for `.gitkeep`.

## evidence.jsonl

`evidence.jsonl` is generated by:

```bash
python scripts/normalize_evidence.py
```

It is ignored by git and should be regenerated from `raw/raw_posts.jsonl`.

## gold_tuples.jsonl and gold_event_chains.jsonl

Gold files are generated only after human review:

```bash
python scripts/convert_review_sheets_to_gold.py
python scripts/validate_gold_dataset.py
```

They are ignored by git and should only contain adjudicated formal gold annotations grounded in `evidence.jsonl`.
